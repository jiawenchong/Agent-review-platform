# AI Agent 開發規劃書範本

## 《治理就緒藍圖 — Governance-Ready Agent Blueprint (GRAB)》 v1.0

> **用途**:這是一份**可重複使用的範本**,給任何要開發新 AI Agent 的提案人填寫。
> 它的設計目標有三個:
>
> 1. **可被 AI 評審中心判讀** — 章節結構固定(§1–§6 為必填),系統會據此給出 🟢綠燈 / 🔴紅燈 / 🟡待補件。
> 2. **可自主產出流程圖** — §7「流程定義」區塊採用固定語法,上傳後後端會**自動生成 Mermaid 流程圖**(`structured` 模式);若未填,系統會依章節標題**自動推斷**一張流程圖(`inferred` 模式)。
> 3. **治理就緒** — §8 紅線、§9 規則/LLM 分工為內建治理骨架,沿用「規則先篩、LLM 後審」原則。
>
> 填寫時請**保留章節編號與標題**,把方括號 `[…]` 的提示換成實際內容即可。

## 七、流程定義（Flow Definition）★驅動自動流程圖

> 本區塊採用固定語法,**系統會自動解析並生成 Mermaid 流程圖**。
> 每一行一個節點,格式如下:
>
> ```
> <節點ID> [<類型>] <標籤> -> <下一節點>
> ```
>
> - **節點ID**:英數字,如 `S1`、`S2`。
> - **類型**:`start`(開始)、`end`(結束)、`process`(處理)、`decision`(判斷)、`io`(輸入/輸出)、`subprocess`(子流程)。
> - **轉移**:`-> S2` 單一去向;判斷節點用 `-> 是:S4 | 否:S9` 標註分支標籤。
> - 至少 2 行才會以 `structured` 模式精準生成;否則系統改用章節標題自動推斷。

**範例(可直接修改):**

```
S1 [start] 開始
S2 [process] 接收上傳規劃書 -> S3
S3 [decision] 文件解析成功? -> 是:S4 | 否:S9
S4 [process] AI 評審中心判讀 -> S5
S5 [decision] 評審結果? -> 綠燈:S6 | 紅燈:S9
S6 [process] 同步至 KANBAN -> S7
S7 [process] 進入每 7 天進度監控（Closed Loop） -> S8
S8 [end] 完成
S9 [end] 退件 · 要求重新提交
```

> 預覽流程圖:`POST /api/uploads/flowchart-preview`,body `{"text": "<上面的流程定義>"}`,
> 即可拿到 Mermaid 定義;上傳整份規劃書時會自動附在解析結果中。

## 八、紅線設定（Guardrails）★必填

逐條對應風險點(可增減,但至少涵蓋資料可得性、語意接地、目標純粹性、權限隔離):

| Guardrail | 觸發條件 | 攔截點 / 處置 |
| --- | --- | --- |
| Capability | `[資料來源失敗時]` | 標示「無法取得資料」,禁止憑空生成 |
| Grounding | `[LLM 引用未驗證資訊]` | 僅可引用 RAG context 與原文,需附來源 |
| Hallucination | `[數據與留言矛盾]` | 標示衝突旗標,不可自行調和 |
| Goal | `[分級受職級/人情影響]` | 僅依治理規範計算,不讀職級欄位 |
| Information Isolation | `[非授權者存取]` | ACL 先檢查,未授權直接拒絕 |
| Escalation | `[合理但連續未解除]` | 升級人工主管,停用 AI 自動核准 |

| 任務 | 方式 | 理由 |
