輸出規則（必須嚴格遵守）：
1. 只輸出 Mermaid `flowchart TD` 程式碼，不要任何說明文字，不要 ``` 圍欄
2. 節點數量 6–12 個，讓圖清晰可讀
3. 起始節點用 `([開始])` 形狀，結束節點用 `([結束])` 或業務意義的終態
4. 節點形狀規則：
   - `([文字])` = 開始 / 終止
   - `[文字]` = 一般步驟（處理、動作）
   - `{文字}` = 判斷 / 決策（需 ≥2 分支）
   - `[/文字/]` = 輸入 / 輸出
5. 決策節點必須有 ≥2 條帶標籤的分支，例如 `-->|是|` / `-->|否|`
6. 節點 ID 用英數字，不能含空格，例如 N1、N2 或 SCAN、JUDGE
7. 標籤用中文，每個節點 ≤12 字
8. 若規劃書有「流程定義」段落（格式：`S1 [process] 標籤 -> S2`），以該段落為優先依據；否則從目標、架構、時程章節推斷業務流程
